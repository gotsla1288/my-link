## 1. 프로젝트 개요

### 1.1 프로젝트명

| 항목 | 내용 |
|------|------|
| 서비스명 | **마이링크 (MyLink)** |
| 레포지토리명 | `my-link` |
| 서비스 URL 패턴 | `mylink.io/@{username}` (예: `mylink.io/@hyerim`) |

### 1.2 목적

링크트리(Linktree)에서 영감을 받은 **개발자·크리에이터 특화 프로필 링크 허브 서비스**입니다. **GitHub, 기술 블로그, YouTube, 포트폴리오 사이트** 등 다수의 링크를 단 하나의 URL로 통합하여 공유할 수 있으며, 기술 스택 위젯·인터랙티브 요소 등 **개발자 브랜딩에 최적화된 기능**을 제공합니다.

단순한 링크 나열을 넘어, **shadcn/ui 기반의 일관된 디자인 시스템**으로 구성된 커스터마이징 기능을 통해 사용자의 개성과 브랜드 아이덴티티를 표현하는 것이 차별점입니다.

### 1.3 배경 및 문제 정의

- 개발자·크리에이터는 GitHub, 기술 블로그, LinkedIn, 유튜브 등 계정이 많지만 SNS 프로필에 링크를 하나만 걸 수 있는 플랫폼이 대부분.
- 기존 링크트리는 무료 플랜의 커스터마이징이 제한적이며, 개발자 특화 위젯(기술 스택, 프로젝트 쇼케이스 등)이 없음.
- 퍼스널 웹사이트를 직접 제작·유지하기에는 비용과 시간이 부담스럽고, 간단한 링크 허브로는 개성 표현에 한계가 있음.

### 1.4 서비스 범위 (v1.0 기준)

- **In-Scope**: 프로필 페이지 생성 및 링크 관리, 관리자 대시보드, 기본 테마 커스터마이징, 링크 클릭 통계
- **Out-of-Scope**: 팀 계정, 링크 판매/결제 기능, 자체 도메인 연결 (v2 이후 검토)

## 3. 대상 사용자 (User Persona)

> **핵심 타겟**: 개발자 / 테크 크리에이터를 1순위 페르소나로 설정하며, 이들의 니즈를 우선 충족하는 방향으로 기능을 설계합니다.

### Persona A — 프론트엔드 개발자 ⭐ 주 타겟

- **이름**: 전혜림 (28세, 프론트엔드 개발자)
- **목표**: GitHub, 기술 블로그, LinkedIn, 포트폴리오 사이트를 한 곳에 정리하고 채용 담당자·협업 파트너에게 공유
- **불만**: 기존 링크트리는 디자인이 단조롭고, 기술 스택 같은 개발자 전용 정보를 표현할 방법이 없음
- **요구사항**: 기술 스택 위젯, 프로젝트 쇼케이스 링크, 깔끔하고 개성 있는 UI
- **Google 계정 보유**: ✅ (Gmail / Google Workspace 사용자)

### Persona B — 테크 유튜버 / 개발 크리에이터 ⭐ 주 타겟

- **이름**: 김도현 (31세, 개발 유튜버 / 오픈소스 기여자)
- **목표**: 유튜브 채널, GitHub 스폰서, 강의 링크, 뉴스레터를 하나의 URL로 팬과 구독자에게 공유
- **불만**: 링크 반응 분석이 부족하고, 각 채널의 통계를 한눈에 보기 어려움
- **요구사항**: 링크별 클릭 통계, 인터랙티브 위젯, 미니멀하고 세련된 디자인
- **Google 계정 보유**: ✅ (YouTube 운영)

### Persona C — 디자이너 / 크리에이티브 프리랜서 (세컨더리)

- **이름**: 박지윤 (30세, UI/UX 디자이너 겸 프리랜서)
- **목표**: 포트폴리오, Behance, 협업 문의 채널을 하나로 묶어 클라이언트에게 전달
- **불만**: 퍼스널 사이트 제작·유지 부담이 크고, 링크트리는 디자인 커스터마이징이 제한적
- **요구사항**: 간단한 관리 UI, 이미지 링크 카드, 테마 커스터마이징
- **Google 계정 보유**: ✅ (Google Drive / Gmail 사용)

## 4. 기술 스택

| 영역 | 기술 | 비고 |
|------|------|------|
| 프레임워크 | Next.js 16 (App Router) | RSC 기반 SSR |
| 언어 | TypeScript | strict 모드 |
| UI 스타일링 | Tailwind CSS v4 | — |
| 디자인 시스템 | **shadcn/ui** | 컴포넌트 기반 일관된 디자인 시스템 |
| 런타임 | React 19 | — |
| 인증 | **Firebase Authentication** | Google OAuth 전용 |
| DB / BaaS | **Firebase Firestore** | 실시간 DB (향후 Supabase 전환 검토) |
| 스토리지 | **Firebase Storage** | 프로필 이미지 업로드 |
| 배포 | Vercel | — |

> **shadcn/ui 선택 이유**: 컴포넌트를 직접 소유(copy-paste)하는 방식으로 디자인 커스터마이징 자유도가 높으며, Radix UI 기반으로 접근성(a11y)이 우수합니다. 이후 디자인 시스템 토큰 교체 시 Tailwind CSS v4 변수와 연동하여 테마 전환이 용이합니다.
>
> **Firebase 선택 이유**: Google OAuth 인증과 Firestore의 실시간 데이터 동기화를 빠르게 구현할 수 있으며, 별도 백엔드 서버 없이 v1.0을 운영할 수 있어 초기 개발 속도가 빠릅니다.

## 6. 기능 상세 설명

### F-01 · 프로필 페이지 (공개 뷰)

**개요**  
`/@{username}` 경로로 접근 가능한 공개 프로필 페이지. 방문자는 로그인 없이 열람 가능.

**화면 구성 요소**

| 섹션 | 내용 |
|------|------|
| 헤더 영역 | 프로필 이미지, 사용자 이름, 한 줄 소개 |
| 링크 목록 | 등록된 링크 버튼 (아이콘 + 레이블 + URL) |
| 위젯 영역 | 선택적으로 표시되는 인터랙티브 위젯 |
| 푸터 | "마이링크로 만들기" CTA 배지 |

**세부 요구사항**
- 모바일 우선(Mobile-First) 반응형 디자인 적용
- 링크 버튼 클릭 시 새 탭에서 열기 (`target="_blank"`)
- 페이지 로딩 시 Skeleton UI로 로딩 상태 표시
- OG 메타태그 자동 설정 (공유 시 미리보기 노출)

---

### F-02 · 링크 추가 / 수정 / 삭제

**개요**  
사용자가 자신의 프로필에 노출할 링크를 CRUD 관리하는 기능.

**입력 필드**

| 필드 | 필수 여부 | 설명 |
|------|----------|------|
| 링크 제목 | 필수 | 버튼에 노출되는 텍스트 (최대 50자) |
| URL | 필수 | `http://` 또는 `https://` 포함 필수, 유효성 검사 |
| 아이콘 | 선택 | 제공 아이콘 세트에서 선택 또는 미지정 |
| 설명 | 선택 | 링크 아래 표시되는 서브텍스트 (최대 100자) |
| 활성화 여부 | 필수 | 공개 프로필에 표시 / 숨김 토글 |

**제약 사항**
- 무료 플랜: 링크 최대 **10개**
- 유료 플랜: 링크 최대 **무제한**
- URL 중복 등록 시 경고 알림 (강제 차단은 하지 않음)

---

### F-03 · 프로필 정보 편집

**개요**  
사용자의 공개 프로필에 표시될 기본 정보를 관리하는 기능.

**편집 가능 항목**

| 항목 | 제약 |
|------|------|
| 프로필 이미지 | JPG / PNG / WebP, 최대 5MB, 정사각형 크롭 |
| 표시 이름 | 최대 30자 |
| 유저네임 (`@username`) | 영문 소문자, 숫자, 언더스코어(`_`), 최대 20자, 중복 불가 |
| 한 줄 소개 | 최대 150자 |
| 카테고리 태그 | 예: `#개발자`, `#디자이너`, `#크리에이터` 등 선택형 |

### F-04 · Google 소셜 로그인 (Firebase Auth)

**개요**  
**Firebase Authentication**을 기반으로 Google 계정 하나로 즉시 가입 및 로그인. 별도 이메일 인증 없이 단 1클릭으로 시작 가능.

**인증 공급자 (v1.0)**

| 공급자 | 상태 | 비고 |
|--------|------|------|
| Google | ✅ **지원** | Firebase Google Provider |
| GitHub | ❌ v1.0 미지원 | v2.0 검토 |
| Kakao | ❌ v1.0 미지원 | v2.0 검토 |

**Firebase Auth 구현 세부사항**
- Firebase SDK `signInWithPopup(googleProvider)` 방식 사용
- Next.js App Router에서 서버·클라이언트 양쪽 세션 처리 (`firebase-admin` SDK 활용)
- 인증 상태는 Firebase `onAuthStateChanged`로 실시간 감지
- ID Token은 HttpOnly 쿠키로 저장하여 서버 컴포넌트에서 검증 가능하도록 처리

**사용자 플로우**
1. 랜딩 페이지에서 "Google로 시작하기" 버튼 클릭
2. Google 계정 선택 팝업 → 동의
3. 최초 로그인 시 유저네임 설정 온보딩 화면
4. 대시보드로 이동

**세부 요구사항**
- 최초 로그인 시 유저네임 설정 온보딩 단계 진행 (중복 실시간 검사)
- Google 계정의 프로필 사진 · 표시 이름 자동 가져오기 (수정 가능)
- 세션 유지: Firebase ID Token 자동 갱신 (1시간 주기)
- 계정 탈퇴 시 Firebase Auth 계정 + Firestore 모든 데이터 + Storage 이미지 즉시 삭제

### F-05 · 링크 순서 드래그 앤 드롭 정렬

**개요**  
관리자 대시보드에서 링크 순서를 마우스/터치 드래그로 변경.

**세부 요구사항**
- 드래그 중 다른 링크가 밀리는 애니메이션 피드백 제공
- 변경된 순서는 실시간 저장 (Optimistic Update 적용)
- 모바일 터치 이벤트 지원 (긴 탭 후 드래그)

**사용 라이브러리 후보**
- `@dnd-kit/core` (권장 — 접근성, 터치 지원 우수)

### F-06 · 기본 테마 선택 (Light / Dark)

**개요**  
프로필 페이지의 전체적인 배색 테마를 선택하는 기능.

**v1.0 제공 테마**

| 테마 ID | 이름 | 설명 |
|---------|------|------|
| `ocean-light` | Ocean Light | 밝은 하늘색 + 흰 배경 (기본값) |
| `ocean-dark` | Ocean Dark | 짙은 슬레이트 + 청량 블루 포인트 |
| `minimal-light` | Minimal Light | 순백 배경, 모노톤 구성 |
| `minimal-dark` | Minimal Dark | 순흑 배경, 흰 텍스트 |

**세부 요구사항**
- 방문자는 프로필 소유자의 설정 테마를 봄 (방문자가 변경 불가)
- 관리자는 미리보기 모드에서 실시간으로 테마 변경 효과 확인 가능

---

### F-07 · 관리자 대시보드

**개요**  
로그인한 사용자가 자신의 프로필과 링크를 관리하는 전용 관리 화면.

**구성 탭**

| 탭 | 기능 |
|----|------|
| 프로필 | 기본 정보 편집 (F-03) |
| 링크 | 링크 추가/수정/삭제/정렬 (F-02, F-05) |
| 외관 | 테마 및 스타일 설정 (F-06, F-11) |
| 통계 | 조회수 및 클릭 통계 (F-08, F-12) |
| 설정 | 계정 설정, 탈퇴, 플랜 관리 |

**세부 요구사항**
- 우측 패널에 실시간 프로필 미리보기 제공
- 변경 사항 자동 저장 (Auto-save) 또는 명시적 "저장" 버튼 제공
- 모바일에서도 접근 가능한 반응형 레이아웃

---

### F-08 · 링크 클릭 수 카운트

**개요**  
방문자가 링크를 클릭할 때마다 해당 링크의 클릭 수를 누적 집계.

**집계 데이터**

| 항목 | 설명 |
|------|------|
| 총 클릭 수 | 링크별 누적 클릭 수 |
| 총 프로필 조회 수 | 프로필 페이지 방문 횟수 |
| 날짜별 클릭 수 | 일/주/월 단위 집계 |

**세부 요구사항**
- 동일 방문자의 중복 클릭은 **1시간 내 1회**로 처리 (IP + User-Agent 기준)
- 소유자 본인의 클릭은 통계에서 제외
- 무료 플랜은 최근 **7일** 데이터만 조회 가능

---

### F-09 · 인터랙티브 위젯

**개요**  
단순 링크 목록을 넘어, 방문자와 소통할 수 있는 인터랙티브 요소를 제공.

**v1.0 제공 위젯**

| 위젯 ID | 이름 | 설명 |
|---------|------|------|
| `W-01` | 파도 보내기 🌊 | 방문자가 버튼을 눌러 응원 에너지를 전송. 클릭 수 누적 및 게이지 표시 |
| `W-02` | 방명록 | 방문자가 짧은 메시지(최대 200자)를 남기는 방명록 기능 |

**파도 보내기 위젯 세부 요구사항**
- 클릭 시 시각적 애니메이션 (수위 상승) 및 클릭 카운터 표시
- 클릭 수는 서버에 저장되어 새로고침 후에도 유지
- 소유자는 대시보드에서 총 파도 수 확인 가능

---

### F-10 · 링크 카테고리 / 그룹핑

**개요**  
링크를 카테고리로 묶어 구조화된 프로필을 구성하는 기능.

**예시 카테고리**
- `SNS`, `포트폴리오`, `블로그`, `연락처`, `쇼핑`, `기타`

**세부 요구사항**
- 카테고리 헤더가 프로필 페이지에 구분선과 함께 표시
- 카테고리 이름은 사용자가 자유 입력 가능 (최대 20자)
- 카테고리도 드래그 앤 드롭 순서 변경 가능

---

### F-11 · 커스텀 컬러 테마 (유료)

**개요**  
기본 테마 이외에 배경색, 버튼 색상, 텍스트 색상을 자유롭게 지정하는 기능.

**커스터마이징 항목**
- 배경 색상 또는 그라디언트 방향 + 색상 2가지
- 링크 버튼 배경색 / 텍스트색 / 테두리색
- 프로필 텍스트 색상
- 폰트 선택 (Google Fonts 제공 10종 이내)

---

### F-12 · 링크별 상세 통계 (유료)

**개요**  
각 링크의 클릭 추이와 방문자 정보를 상세하게 분석하는 대시보드.

**제공 데이터**

| 지표 | 설명 |
|------|------|
| 클릭 추이 그래프 | 일/주/월 단위 클릭 수 라인 차트 |
| 유입 기기 | 모바일 / 데스크톱 / 태블릿 비율 |
| 국가 / 지역 | 방문자 접속 국가 (Top 5) |
| 참조 페이지 | 어떤 경로에서 유입되었는지 (Referrer) |

---

### F-13 · 커스텀 배경 이미지 (유료)

**개요**  
프로필 페이지 배경에 이미지를 업로드하거나 URL을 지정하는 기능.

**세부 요구사항**
- 이미지 업로드: JPG / PNG / WebP, 최대 10MB
- 오버레이 투명도 조절 가능 (0~80%)
- 배경 이미지 위에서 텍스트 가독성을 위한 자동 대비 조정 권고

---

### F-14 · 이미지 링크 카드 (유료)

**개요**  
링크 버튼 대신 썸네일 이미지가 포함된 카드 형태로 링크를 표시.

**세부 요구사항**
- 직접 이미지 업로드 또는 URL 입력 시 OG 이미지 자동 수집
- 카드 레이아웃: 가로형 또는 정방형 선택 가능
- 설명 텍스트 최대 2줄 표시 (말줄임 처리)

---

### F-15 · SNS OG 미리보기 자동 수집 (유료)

**개요**  
링크 URL을 입력하면 해당 페이지의 OG 태그(제목, 설명, 이미지)를 자동으로 불러와 링크 정보를 자동 완성.

**처리 흐름**
1. 사용자가 URL 입력 후 일정 시간 대기 (Debounce)
2. 서버에서 URL의 OG 태그 크롤링
3. 제목·설명·썸네일을 미리보기로 표시
4. 사용자가 자동 수집 정보를 확인 후 수정 가능

---

### F-16 · 링크 예약 공개 / 비공개 토글 (유료)

**개요**  
특정 링크를 지정된 날짜/시간 이후에 자동으로 공개하거나 비공개로 전환하는 예약 기능.

**세부 요구사항**
- 공개 시작일시 / 공개 종료일시 각각 설정 가능
- 비공개 상태의 링크는 방문자 화면에서 완전히 숨김 처리
- 예약 시각은 사용자 기준 로컬 타임존으로 입력 및 표시

---

### F-17 · QR 코드 다운로드

**개요**  
자신의 프로필 URL에 대한 QR 코드를 생성하여 PNG 파일로 다운로드.

**세부 요구사항**
- QR 코드 사이즈 선택: 256px / 512px / 1024px
- 중앙에 프로필 이미지 삽입 옵션 (선택)
- 파일명 형식: `mylink-@{username}-qr.png`

---

### F-18 · 기술 스택 위젯 (개발자용)

**개요**  
개발자 페르소나를 위한 전용 위젯. 사용하는 기술 스택을 배지 형태로 프로필에 표시.

**세부 요구사항**
- 제공 기술 태그 풀: React, Next.js, TypeScript, Python, Go, AWS 등 50개 이상
- 숙련도 레이블 선택: `Beginner / Intermediate / Expert`
- 최대 12개 기술 등록 가능

## 8. 제약사항 및 가정

### 제약사항
- 초기 v1.0은 **한국어 서비스**만 지원 (i18n은 v2 이후)
- 결제 시스템 연동은 v1.0 범위 밖 (Toss Payments 연동 v2 예정)
- 외부 크롤링(OG 수집 등)은 서버사이드에서만 처리

### 가정
- 사용자는 스마트폰 또는 PC 브라우저를 통해 접속한다
- 사용자는 소셜 계정(Google, GitHub, Kakao) 중 하나 이상을 보유한다
- 링크 URL은 사용자가 직접 검증하여 입력한다

## 💡 개선 제안 (Agent Notes)

현재 PRD 작성 과정에서 다음 사항을 추가 검토하면 좋을 것 같아 별도로 기록합니다.

1. **애널리틱스 데이터 보관 정책 명시**: GDPR / 개인정보보호법 관련, 수집하는 방문자 데이터(IP 등)의 보관 기간과 익명화 처리 방침을 정책으로 명시하는 것을 권장합니다.

2. **스팸 / 악성 링크 방지**: 피싱·악성 URL이 등록될 경우를 대비하여, 등록 URL에 대한 Google Safe Browsing API 연동 또는 신고 시스템 도입을 초기부터 설계에 포함하는 것을 권장합니다.

3. **링크 미리보기 캐싱 전략**: OG 태그를 수집하는 F-15의 경우, 동일 URL에 대한 반복 크롤링을 막기 위한 캐시 만료 정책(예: TTL 24시간)을 설계 단계에서 결정해야 합니다.

4. **유저네임 예약 시스템**: `admin`, `api`, `static` 등 서비스 내부 경로와 충돌하는 유저네임을 블랙리스트로 관리하는 로직이 필요합니다.

5. **소셜 공유 미리보기 테스트 도구**: 관리자 대시보드에 카카오톡, 트위터, 슬랙 등 주요 플랫폼에서 링크 공유 시 어떻게 보이는지 미리보기할 수 있는 도구를 제공하면 사용자 경험이 크게 향상됩니다.
