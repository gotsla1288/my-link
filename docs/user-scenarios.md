## 1. 문서 개요

### 1.1 작성 목적

이 문서는 마이링크 서비스를 이용하는 두 가지 주요 사용자 유형의 행동 흐름을 시나리오 형식으로 정리합니다.  
각 시나리오는 다음 형식을 따릅니다.

> **"사용자는 `[목적]`하기 위해 `[행동]`한다."**

### 1.2 사용자 유형

| 구분 | 설명 |
|------|------|
| **방문자** | 로그인 없이 타인의 프로필 페이지를 열람하는 사용자 |
| **소유자** | Google 계정으로 로그인하여 자신의 프로필과 링크를 직접 관리하는 사용자 |

---

## 2. 방문자 시나리오

> 방문자는 로그인 없이 `/@{displayName}` 경로에 접근하며, 프로필 열람과 링크 클릭이 주요 행동입니다.

---

### SC-V-01 · 프로필 페이지 접속

방문자는 **소유자의 마이링크 페이지를 열람**하기 위해 공유된 URL(예: `mylink.io/@hyerim`)을 브라우저 주소창에 입력하거나 링크를 클릭한다.

방문자는 **페이지 로딩 상태를 인지**하기 위해 콘텐츠가 불러와지는 동안 표시되는 Skeleton UI를 확인한다.

방문자는 **소유자의 기본 정보를 파악**하기 위해 페이지 상단의 프로필 이미지, 유저네임, 한 줄 소개(bio)를 읽는다.

---

### SC-V-02 · 링크 클릭

방문자는 **소유자가 공유한 링크에 접근**하기 위해 프로필 페이지에 나열된 링크 버튼 중 원하는 항목을 클릭한다.

방문자는 **현재 페이지를 유지한 채 링크를 확인**하기 위해 새 탭으로 열리는 외부 URL 페이지를 이용한다.

---

### SC-V-03 · 파도 보내기 (F-09)

방문자는 **소유자에게 응원 메시지를 전달**하기 위해 프로필 페이지의 파도 보내기 위젯 버튼을 클릭한다.

방문자는 **자신의 파도 전송이 반영되었음을 확인**하기 위해 버튼 클릭 후 재생되는 수위 상승 애니메이션과 증가한 카운터 숫자를 확인한다.

---

### SC-V-04 · 마이링크 서비스 인지

방문자는 **마이링크 서비스를 알게 되어 자신도 사용해 보기 위해** 프로필 페이지 하단의 "마이링크로 만들기" CTA 배지를 클릭하여 랜딩 페이지로 이동한다.

---

## 3. 소유자 시나리오

> 소유자는 Google 계정으로 로그인한 후 관리자 대시보드를 통해 프로필과 링크를 관리합니다.

---

### 3.1 최초 가입 및 온보딩 (F-04)

---

#### SC-O-01 · Google 소셜 로그인

소유자는 **마이링크 서비스에 처음 가입**하기 위해 랜딩 페이지의 "Google로 시작하기" 버튼을 클릭한다.

소유자는 **Google 계정으로 인증을 완료**하기 위해 팝업으로 표시된 Google 계정 선택 화면에서 사용할 계정을 선택하고 동의한다.

---

#### SC-O-02 · 온보딩 — 프로필 초기 설정

소유자는 **자신의 프로필 URL 슬러그(displayName)를 확인**하기 위해 온보딩 화면에서 Gmail ID 앞부분으로 자동 설정된 디스플레이 네임을 확인한다.

소유자는 **원하는 URL 주소를 사용**하기 위해 자동 설정된 디스플레이 네임이 마음에 들지 않을 경우 직접 수정 입력하고, 실시간으로 표시되는 중복 여부 메시지를 확인한다.

소유자는 **대시보드 진입을 완료**하기 위해 온보딩 화면에서 "확인" 버튼을 눌러 관리자 대시보드로 이동한다.

---

### 3.2 프로필 정보 편집 (F-03)

---

#### SC-O-03 · 프로필 이미지 변경

소유자는 **공개 프로필에 표시될 사진을 교체**하기 위해 대시보드 프로필 탭에서 현재 프로필 이미지 영역을 클릭하여 기기 내 사진 파일을 업로드한다.

소유자는 **이미지가 올바르게 적용되었는지 확인**하기 위해 업로드 후 미리보기 화면에서 크롭된 결과를 검토한다.

---

#### SC-O-04 · 유저네임(실제 이름) 수정

소유자는 **프로필 카드에 표시되는 이름을 변경**하기 위해 대시보드 프로필 탭의 유저네임 입력 필드를 수정하고 저장한다.

---

#### SC-O-05 · 디스플레이 네임(URL 슬러그) 변경

소유자는 **공유 URL 주소를 변경**하기 위해 대시보드 프로필 탭의 디스플레이 네임 필드를 수정한다.

소유자는 **URL 변경으로 인한 영향을 인지**하기 위해 변경 저장 전 표시되는 "기존 URL이 무효화될 수 있습니다" 안내 메시지를 확인하고 저장한다.

---

#### SC-O-06 · 한 줄 소개(bio) 작성

소유자는 **방문자에게 자신을 소개**하기 위해 대시보드 프로필 탭에서 한 줄 소개(bio) 필드에 텍스트를 입력하고 저장한다.

---

### 3.3 링크 관리 (F-02)

---

#### SC-O-07 · 링크 추가

소유자는 **방문자에게 새로운 링크를 노출**하기 위해 대시보드 링크 탭에서 "링크 추가" 버튼을 클릭한다.

소유자는 **링크 정보를 입력**하기 위해 링크 추가 폼에서 링크 제목(필수), URL(필수), 아이콘(선택), 설명(선택), 활성화 여부를 설정한다.

소유자는 **유효한 URL만 등록**하기 위해 잘못된 형식의 URL 입력 시 표시되는 유효성 검사 오류 메시지를 확인하고 수정한다.

소유자는 **링크 저장을 완료**하기 위해 폼 작성 후 "저장" 버튼을 클릭하여 링크 목록에 추가된 결과를 확인한다.

> **참고**: 무료 플랜 사용자는 최대 10개까지 등록 가능하며, 한도 초과 시 안내 메시지가 표시됩니다.

---

#### SC-O-08 · 링크 수정

소유자는 **기존에 등록한 링크의 정보를 업데이트**하기 위해 대시보드 링크 탭에서 수정할 링크의 편집 버튼을 클릭한다.

소유자는 **변경 사항을 반영**하기 위해 링크 제목, URL, 아이콘, 설명 중 원하는 항목을 수정하고 "저장" 버튼을 클릭한다.

---

#### SC-O-09 · 링크 비활성화(숨김)

소유자는 **특정 링크를 공개 프로필에서 임시로 숨기기** 위해 대시보드 링크 탭에서 해당 링크의 활성화 토글을 비활성 상태로 전환한다.

---

#### SC-O-10 · 링크 삭제

소유자는 **더 이상 필요하지 않은 링크를 제거**하기 위해 대시보드 링크 탭에서 삭제할 링크의 삭제 버튼을 클릭한다.

소유자는 **실수로 인한 삭제를 방지**하기 위해 표시되는 삭제 확인 다이얼로그에서 "삭제" 버튼을 클릭하여 최종 삭제를 확정한다.

---

#### SC-O-11 · 링크 순서 재정렬 (F-05)

소유자는 **공개 프로필에 표시되는 링크 순서를 조정**하기 위해 대시보드 링크 탭에서 원하는 링크 항목을 마우스(또는 터치)로 드래그하여 원하는 위치에 놓는다.

소유자는 **재정렬이 저장되었음을 확인**하기 위해 드롭 직후 자동 저장 처리되는 결과를 링크 목록에서 확인한다.

---

### 3.4 테마 설정 (F-06)

---

#### SC-O-12 · 테마 선택

소유자는 **자신의 프로필 페이지 분위기를 원하는 스타일로 변경**하기 위해 대시보드 외관 탭에서 제공되는 테마 목록(Ocean Light / Ocean Dark / Minimal Light / Minimal Dark) 중 하나를 선택한다.

소유자는 **테마 변경 결과를 미리 확인**하기 위해 테마 선택 즉시 갱신되는 미리보기 화면에서 실제 적용 모습을 검토한다.

소유자는 **선택한 테마를 공개 프로필에 반영**하기 위해 미리보기 확인 후 "저장" 버튼을 클릭한다.

---

### 3.5 계정 관리

---

#### SC-O-13 · 로그아웃

소유자는 **현재 기기에서 세션을 종료**하기 위해 대시보드 설정 탭 또는 상단 네비게이션의 로그아웃 메뉴를 클릭한다.

---

#### SC-O-14 · 계정 탈퇴

소유자는 **마이링크 서비스 이용을 완전히 중단**하기 위해 대시보드 설정 탭에서 "계정 탈퇴" 버튼을 클릭한다.

소유자는 **탈퇴로 인한 데이터 삭제를 인지**하기 위해 표시되는 안내 메시지("모든 프로필, 링크, 이미지 데이터가 즉시 삭제됩니다")를 확인하고 탈퇴 확인 버튼을 클릭한다.

## 💡 개선 제안

PRD 검토 중 발견한 추가 시나리오 후보입니다. 필요에 따라 포함 여부를 결정해 주세요.

| 제안 ID | 내용 | 관련 기능 |
|---------|------|----------|
| **제안 1** | **중복 URL 등록 경고 처리 시나리오** — PRD에 "URL 중복 등록 시 경고 알림(강제 차단은 하지 않음)"이 명시되어 있으나, 사용자가 경고를 인지하고 어떻게 대응하는지 시나리오가 필요합니다. | F-02 |
| **제안 2** | **링크 한도 초과 시나리오** — 무료 플랜 10개 초과 시도 시 사용자 흐름이 필요합니다. 단순 에러 표시인지, 업그레이드 유도 CTA인지 명확히 해야 합니다. | F-02 |
| **제안 3** | **displayName 중복 충돌 시나리오** — 온보딩 외에도 설정 탭에서 변경 시 중복 충돌이 발생할 경우의 시나리오가 필요합니다. | F-03, F-04 |
| **제안 4** | **존재하지 않는 URL 접근 시나리오** — `/@없는유저` 접근 시 404 페이지 흐름이 방문자 시나리오에 포함되면 좋습니다. | F-01 |
| **제안 5** | **통계 확인 시나리오** — F-08(링크 클릭 수 카운트)이 PRD 필수 기능인데, 소유자가 대시보드에서 통계를 확인하는 시나리오가 빠져 있습니다. 포함할지 확인 부탁드립니다. | F-08 |
