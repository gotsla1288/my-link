## 1. 화면 구성 개요

방문자용 프로필 페이지(`/@{displayName}`)는 **모바일 우선(Mobile-First)** 단일 컬럼 레이아웃으로 구성됩니다.

| 영역 | 구성 요소 | 비고 |
|------|-----------|------|
| **헤더** | 프로필 이미지 · 유저네임 · 한 줄 소개 | 항상 표시 |
| **링크 목록** | 링크 버튼 카드 (아이콘 + 제목 + 설명) | 등록 순서대로 세로 배열 |
| **위젯 영역** | 파도 보내기 위젯 🌊 | 소유자 활성화 시만 표시 |
| **푸터** | "마이링크로 만들기" CTA 배지 | 항상 표시 |

### 7.3 저장 흐름

```mermaid
flowchart TD
    OwnerView(["소유자\n본인 페이지 접속\n(로그인 상태)"])

    OwnerView --> EditBanner["상단 편집 배너 표시\n✏️ 내 페이지 편집 중"]

    EditBanner --> ClickEdit["프로필 항목 옆\n✏️ 버튼 클릭"]

    ClickEdit --> Inline["인라인 편집 필드\n활성화\n(username / bio / 이미지)"]

    Inline -->|"[취소] 클릭"| Cancel["편집 취소\n원래 값 복원"]
    Inline -->|"[저장] 클릭"| Validate{"유효성 검사"}

    Validate -->|"실패\n(빈 값, 초과 등)"| Error["인라인 오류 메시지\n필드 강조 표시"]
    Validate -->|"성공"| Save["Firestore 저장\n+ 화면 즉시 반영"]

    Error --> Inline
    Save --> Success["저장 완료\n토스트 알림 표시"]
    Cancel --> OwnerView
```

### 7.4 소유자 vs 방문자 뷰 비교

| 요소 | 방문자 뷰 | 소유자 뷰 |
|------|----------|----------|
| 상단 배너 | ❌ 없음 | ✅ "편집 중" 배너 표시 |
| 프로필 이미지 | 표시만 | 📷 버튼으로 교체 가능 |
| username | 텍스트만 | ✏️ 버튼 → 인라인 필드 |
| bio | 텍스트만 | ✏️ 버튼 → 인라인 textarea |
| 링크 목록 | 클릭 가능 | 클릭 가능 (편집은 대시보드에서) |
| 링크 추가 단축 버튼 | ❌ 없음 | ✅ "링크 추가 → 대시보드" CTA |
| 파도 위젯 | 파도 보내기 가능 | 파도 수 확인만 (본인 클릭 통계 제외) |
| CTA 배지 | "마이링크로 만들기" | 동일하게 표시 |
